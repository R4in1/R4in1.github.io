一个探究无线网络安全的记录

发表于2021-05-06|更新于2021-05-08|MIX
|字数总计:3k|阅读时长:9分钟|阅读量:

参考了学校图书馆的《Kali Linux无线网络渗透测试详解》

WIFI网络概述

网络按照区域划分，分为局域网、城域网和广域网（细分还可以分更多，不过没有太大的意义）。无线网络一般是相对局域网来说的，平常我们说的WLAN就是无线网络，而WIFI（Wireless Fidelity）是现在最流行的无线网络模式，它是一个建立于IEEE 802.11标准的无线局域网（WLAN）设备标准。

一般架设WIFI网络的基本设备就是无线网卡和一台AP（Access Point，可翻译为无线访问接入点），一般家用的路由器都不会只有路由功能，为了方便移动设备上网都会添加无线覆盖功能，也就是提供WIFI。用户的终端如果安装了无线网卡，就可以通过无线网卡连接到AP实现无线上网功能。

WIFI工作原理

AP启动之后，如果没有设置隐藏SSID（Service Set Identifier）的话，会每100ms将SSID经由beacons（信号台，理解为天线）封包广播一次，因为这个包非常的短，所以这个广播对网络的性能影响并不大。当客户端打开WIFI的时候就可以收到这个SSID广播，并决定是否和这个SSID的AP建立连接。

SSID：Service Set Identifier，服务集标识，SSID技术可以将一个无线局域网划分为几个需要不同的身份验证的子网络，每个子网络都需要独立的身份验证，只有通过对应的子网络的身份验证才能接入相应的子网络，比如迅捷路由器设置访客网络，可以在路由器管理界面设置开启访客网络

然后就可以在WIFI列表里看到两个SSID，两个SSID需要不同的方式进行认证，但是这两个SSID都是同一个AP提供的：

不过一般日常使用的无线路由器都只设置一个SSID

信道：无线信道也就是无线的“频段”（CHannel），在无线路由器中，802.11b/g/n模式（也就是2.4G模式）下通常由有13个信道，每个信道中心频率相差5MHz，有效宽度20MHz。无线网络可以在多个信道上运行，在无线信号覆盖范围内的各种无线网络设备应尽量使用不同的信道以避免相互干扰。

在Kali中配置无线网卡

虚拟机无法直接使用物理机上的无线网卡，必须外接无线网卡，然后在VM中选择将网卡介入到Kali中，然后使用iwconfig就可以看到接入的无线网卡了

无线网卡通常包含四种模式：

  • Managed 模式：该模式用于你的无线客户端直接与无线接入点（Access Point，即AP）进行接入连接。在这个模式中，我们可以进行无线接入internet上网，无线网卡的驱动程序依赖无线AP管理整个通信过程。

  • Master 模式：一些无线网卡支持Master模式。这个模式允许无线网卡使用特制的驱动程序和软件工作，作为其他设备的无线AP。它主要使用于无线接入点AP提供无线接入服务以及路由功能。比如无线路由器就是工作在Master模式下了，对于普通的pc机来说，如果有合适的硬件它也可以变成一台无线AP。

  • Ad hoc模式：当网络由互相直连的设备组成时，就使用这个模式。在这个模式中，各设备之间采用对等网络的方式进行连接，无线通信双方共同承担无线AP的职责。

  • Monitor 模式：Monitor模式主要用于监控无线网络内部的流量，用于检查网络和排错。要使Wireshark捕获无线数据包，无线网卡和配套驱动程序必须支持监听模式（也叫RFMON模式）。

在WIFI网络中，无线网卡是以广播的形式发射信号的，信号发射出去所有的设备都可以接收到该信息，但是只有目的地址与自己的地址一致的时候设备才会接受这个数据包，所以如果要接收整个网络中的所有流量就要将网卡设为Monitor模式，也就是计算机网络中所称的混杂模式。

设置网卡为监听模式

Aircrack-ng是一个与802.11标准的无线网络分析有关的安全软件，主要功能包括网络侦测、数据包嗅探、WEP和WPA/WPA2-PSK破解，可以工作在任何支持监听模式的网卡上。

首先使用ifconfig查看网卡，如果没有看到wlan0，但是使用ifconfig -a可以看到wlan0，则说明没有激活该网卡，使用ifconfig wlan0 up来激活网卡，然后再使用该网卡。同理，使用命令ifconfig wlan0 down可以关闭网卡。

然后使用airmon-ng工具启动无线网卡的监听模式，执行airmon-ng start wlan0来设置网卡为监听模式：

上图中还可以看到网卡使用的芯片为RT5370，驱动为rt2800usb，然后使用ifconfig命令再次查看网络接口，可以看到有一个接口名字为wlan0mon，说明网卡已经启动监听模式：

使用iwconfig命令可以只查看无线网卡，可以看到wlan0mon的模式（Mode）为混杂模式（Montitor）：

然后使用airodump-ng工具可以使用监听模式下的网卡捕捉802.11数据报文，用来扫描附近的AP提供的SSID（包括隐藏的），执行airodump-ng wlan0mon，可以看到如下结果：

其中BSSID表示无线AP的Mac地址。PWR表示无线信号的信号水平，信号值越高说明离AP越近。如果某个BSSID的PWR为-1则说明网卡的驱动不支持报告信号水品。Beacons表示无线AP发出的通告编号，每个AP在最低速率时每秒会发送10个左右的beacon。#Data表示被捕获到的分组数量（如果是WEP，则代表唯一IV的数量，但是现在的WIFI基本没有WEP的了，太不安全了），包括广播分组，#/s表示过去10秒钟每秒捕获的数据分组的数量。CH表示信道号，MB为无线AP所支持的最大速率。ENC表示使用的加密算法体系，OPN表示无加密，WPA2表示WPA/WPA2，CIPHER表示检测到的加密算法，一般来说CCMP与WPA2结合使用。AUTH表示使用的认证协议，PSK表示WPA/WPA2的与预享密钥。ESSID就是所谓的SSID号，也就是WIFI名称，如果为空或者显示<length: 0>，就说明该AP启用了隐藏SSID，airodump-ng会尝试从proberesopnce和associationrequests中获取SSID，也就是尝试在正常用户尝试连接隐藏的SSID的时候获取SSID。

开放的网络

因为现在大多数的路由器都不用WEP了，因为已经被证明不安全了，所以先把实验用的zR4in设置为不加密尝试抓包

在kali中打开wireshark，然后选择刚刚使用工具设为Monitor模式的网络接口，然后就可以开始抓包了，但是实际上尝试了很多次才成功抓到HTTP包，个人猜测是因为附近的WIFI太多了，在空中的数据包实在是太多，有可能与网卡的性能有关，为了缓解这种情况，试了一下直接使用linux的ifconfig和iwconfig来设置网卡，首先将无线网卡接入kali，然后ifconfig可以看到有wlan0，这时候要先用ifconfig wlan0 down把网卡关掉，不然iwconfig设置网卡的时候会提示设备正在忙，关掉之后iwconfig wlan0 mode monitor将网卡设置为monitor模式，然后iwconfig wlan0 channel 13设置网卡使用的信道为13信道，在路由器的管理界面也将路由器设置为13信道，这样可以尽量过滤掉一些其他路由器的干扰，然后ifconfig wlan0 up启动网卡，然后在wireshark中的视图菜单选中无线工具：

然后也将抓包的信道设为13信道：

然后大概前后试了十几次，大概抓了几百万个包，终于成功抓到了HTTP包，数据包的数量多其实主要是因为广播包太多了，因为AP会发出广播信号，当客户端打开WIFI想要接入AP的时候，扫描SSID也会发出广播包。还有很多控制帧，然后网卡性能不太好的话，就导致正常的访问外部网站的数据包抓不到。

分析一下抓到的包，选择将数据包按照Protocol排序，只看上面的TCP和HTTP包（其实HTTP包很少）：

可以看到有HTTP的流量，右键选中追踪HTTP流：

这是我用连接了zR4in这个WIFI的iPad访问一个工具网站的HTTP请求，网站没有启用HTTPS，所以直接就可以看到明文的数据，Uesr-Agent可以看到时iPad使用safari发出的请求，并且看数据包的信息可以知道iPad在局域网中的ip为192.168.1.101，甚至可以直接在HTTP报文中看到明文的cookie，请求发送的post数据也可以，可以说是十分危险了，所以说不要随便连接没有密码的开放WIFI。

可能是因为网卡性能不太好所以只能抓一个信道的包，考虑一下如果网卡和电脑性能足够的话，应该可以抓到所有信道的数据包，窃听所有开放网络的请求。

WEP加密

很多路由器都已经不支持WEP了，我的两个路由器都不支持，所以没办法进行实验，但还是记录一下WEP加密的原理以及缺陷
